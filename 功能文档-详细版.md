## 一、资源分配管理

### 1.1 角色与用户管理

#### 功能描述
提供系统角色和用户的完整管理功能，支持角色权限分配和用户角色分配，确保系统安全性和权限控制。

#### 设计思路

**架构设计**：
- 采用RBAC（基于角色的访问控制）模型，实现用户-角色-权限的三层权限管理
- 用户与角色是多对多关系，通过中间表`sys_user_role`关联
- 角色与菜单权限是多对多关系，通过中间表`sys_role_menu`关联
- 密码采用MD5加盐加密，盐值为用户名，进行3次MD5哈希增强安全性

**核心设计模式**：
- **策略模式**：权限验证时采用策略模式，根据用户角色动态加载权限
- **事务管理**：使用`@Transactional`确保用户-角色关系的原子性操作
- **DTO模式**：使用DTO进行数据传输，避免实体类直接暴露

**数据库设计**：
```
sys_user (用户表)
  ├── id, username, password, name, dept_id, email, phone
  └── is_deleted, create_time, update_time

sys_role (角色表)
  ├── id, name, code, description
  └── is_deleted, create_time

sys_user_role (用户角色关联表)
  ├── id, user_id, role_id
  └── create_time

sys_role_menu (角色菜单关联表)
  ├── id, role_id, menu_id
  └── create_time

sys_menu (菜单表)
  ├── id, name, url, parent_id, type, sort_order
  └── permission, icon, is_deleted
```

**关键业务流程**：
1. 保存角色时，先删除旧的权限关系，再批量插入新的权限关系
2. 保存用户时，先删除旧的角色关系，再批量插入新的角色关系
3. 登录时通过用户名查询用户，再通过用户查询角色，最后通过角色查询菜单权限

#### 核心代码说明

**1. 角色权限分配核心代码**：

```java
// SysRoleServiceImpl.java - rebuildRoleMenu方法
@Override
@Transactional(rollbackFor = Exception.class)
public void rebuildRoleMenu(SysRoleDTO sysRoleDTO) throws Exception {
    // 1. 先删除该角色的所有旧权限关系
    if (StringUtils.isNotEmpty(sysRoleDTO.getId())) {
        QueryWrapper<SysRoleMenu> deleteWrapper = new QueryWrapper<>();
        deleteWrapper.eq("role_id", sysRoleDTO.getId());
        sysRoleMenuService.remove(deleteWrapper);
    }
    
    // 2. 批量插入新的权限关系
    if (ArrayUtils.isNotEmpty(sysRoleDTO.getMenuIds())) {
        for (String menuId : sysRoleDTO.getMenuIds()) {
            if (StringUtils.isEmpty(menuId)) {
                continue;
            }
            SysRoleMenu sysRoleMenu = new SysRoleMenu();
            sysRoleMenu.setRoleId(sysRoleDTO.getId());
            sysRoleMenu.setMenuId(menuId);
            sysRoleMenuService.save(sysRoleMenu);
        }
    }
}
```

**设计要点**：
- 使用`@Transactional`确保操作的原子性，失败时回滚
- 先删除后插入的策略，避免数据不一致
- 使用MyBatis Plus的`QueryWrapper`构建查询条件

**2. 用户保存与密码加密核心代码**：

```java
// SysUserServiceImpl.java - save方法
@Transactional(rollbackFor = Exception.class)
@Override
public void save(SysUserDTO record) throws Exception {
    // MD5算法计算3次，增强安全性
    String result = new Md5Hash(record.getPassword(), record.getUsername(), 3).toString();
    record.setPassword(result);
    sysUserMapper.insert(record);
    // 建立用户角色关系
    sysRoleService.rebuild(record);
}
```

**设计要点**：
- 使用Shiro的`Md5Hash`进行密码加密，盐值为用户名
- 3次MD5哈希增加破解难度
- 密码不在数据库中明文存储

**3. 获取用户权限树核心代码**：

```java
// SysUserServiceImpl.java - getUserAndRoleAndMenuByUsername方法
@Override
public SysUserDTO getUserAndRoleAndMenuByUsername(String username) throws Exception {
    // 1. 查询用户基本信息
    SysUserDTO result = sysUserMapper.selectUserAndRoleByUsername(username);
    
    // 2. 遍历用户的角色，获取每个角色的菜单权限
    if (CollectionUtils.isNotEmpty(result.getSysRoleDTOs())) {
        for (SysRoleDTO rDto : result.getSysRoleDTOs()) {
            List<SysMenuDTO> menus = sysMenuService.listByRoleId(rDto.getId());
            rDto.setSysMenuDtos(menus);
        }
    }
    return result;
}
```

**设计要点**：
- 一次性获取用户-角色-权限的完整树形结构
- 支持用户拥有多个角色，权限取并集

### 1.2 班组员工管理

#### 功能描述
管理生产班组及班组内的员工分配，支持按部门组织员工，形成生产作业团队。

#### 设计思路

**架构设计**：
- 班组是生产作业的基本组织单位，与部门关联
- 班组与员工是多对多关系，通过中间表`sys_group_user`关联
- 支持一个员工属于多个班组（灵活配置）
- 采用"先删除后插入"策略维护班组-员工关系

**数据库设计**：
```
sys_group (班组表)
  ├── id, name, code, dept_id, type, description
  └── is_deleted, create_time, update_time

sys_group_user (班组用户关联表)
  ├── id, group_id, user_id
  └── create_time

sys_department (部门表)
  ├── id, name, code, parent_id, sort_order
  └── is_deleted, create_time
```

**关键业务流程**：
1. 新增班组时，先保存班组基本信息
2. 然后批量保存班组与员工的关联关系
3. 编辑班组时，先删除旧的关联关系，再插入新的关联关系

#### 核心代码说明

**班组员工关系维护核心代码**：

```java
// SysGroupServiceImpl.java - rebuildGroupUser方法
@Override
@Transactional(rollbackFor = Exception.class)
public void rebuildGroupUser(SysGroupDTO sysGroupDTO) throws Exception {
    // 1. 先删除该班组的所有旧成员关系
    if (StringUtils.isNotEmpty(sysGroupDTO.getId())) {
        QueryWrapper<SysGroupUser> deleteWrapper = new QueryWrapper<>();
        deleteWrapper.eq("group_id", sysGroupDTO.getId());
        sysGroupUserService.remove(deleteWrapper);
    }
    
    // 2. 批量插入新的成员关系
    if (ArrayUtils.isNotEmpty(sysGroupDTO.getUserIds())) {
        for (String userId : sysGroupDTO.getUserIds()) {
            if (StringUtils.isEmpty(userId)) {
                continue;
            }
            SysGroupUser sysGroupUser = new SysGroupUser();
            sysGroupUser.setGroupId(sysGroupDTO.getId());
            sysGroupUser.setUserId(userId);
            sysGroupUserService.save(sysGroupUser);
        }
    }
}
```

**设计要点**：
- 使用事务确保数据一致性
- 先删除后插入策略避免脏数据
- 支持批量操作提高效率

### 1.3 生产设备与编组管理

#### 设计思路

**架构设计**：

#### 核心代码说明

**物料编码自动生成核心代码**：

```java
// SpMaterialInfoServiceImpl.java - generateMaterialCode方法
@Override
public String generateMaterialCode() {
    // 1. 获取当前日期作为编码前缀
    String datePrefix = LocalDate.now().format(DateTimeFormatter.ofPattern("yyyyMMdd"));
    
    // 2. 查询当天最大的物料编码
    QueryWrapper<SpMaterialInfo> qw = new QueryWrapper<>();
    qw.likeRight("code", datePrefix);  // LIKE '20240604%'
    qw.orderByDesc("code");
    qw.last("limit 1");
    SpMaterialInfo last = getOne(qw);
    
    // 3. 如果存在当天编码，流水号+1
    if (last != null && last.getCode() != null && last.getCode().startsWith(datePrefix)) {
        String suffix = last.getCode().substring(datePrefix.length());
        int seq = Integer.parseInt(suffix) + 1;
        return datePrefix + String.format("%04d", seq);
    }
    
    // 4. 如果当天没有编码，从0001开始
    return datePrefix + "0001";
}
```

**设计要点**：
- 使用`likeRight`进行前缀匹配，利用索引提高查询效率
- 使用`String.format("%04d", seq)`确保流水号固定4位，不足补0
- 查询结果按编码降序排列，取第一条即为最大编码
- 线程安全考虑：在高并发场景下需要加锁或使用数据库序列

### 2.2 物料需求计划

#### 功能描述
根据生产订单自动计算物料需求，支持一键生成入库单，登账后自动更新库存。

#### 设计思路

**架构设计**：
- MRP（物料需求计划）是连接生产订单和物料采购的桥梁
- 通过BOM展开计算物料需求，考虑损耗率和现有库存
- MRP记录与入库单关联，支持一键生成入库单
- 入库单登账后自动更新库存，保证数据一致性

**MRP计算逻辑**：
```
毛需求 = 订单数量 × BOM用量 × (1 + 损耗率)
净需求 = 毛需求 - 现有库存 - 在途库存
```

**数据库设计**：
```
sp_mrp_record (MRP需求记录表)
  ├── id, mrp_no, order_code, bom_code
  ├── product_code, product_name
  ├── part_code, part_name, demand_qty, unit
  ├── source_order_no
  └── is_deleted, create_time, update_time

sp_inbound_order (入库单表)
  ├── id, inbound_no, source_mrp_nos
  ├── status, item_count, total_demand_qty
  ├── warehouse_id, warehouse_location_id, warehouse_location_ids
  └── is_deleted, create_time, update_time

sp_inbound_order_item (入库单明细表)
  ├── id, inbound_order_id, source_mrp_record_id
  ├── mrp_no, order_code, bom_code
  ├── product_code, product_name, part_code, part_name
  ├── demand_qty, unit
  └── is_deleted, create_time
```

**关键业务流程**：
1. 生产订单执行时，根据BOM自动生成MRP记录
2. 在MRP列表中选择需要入库的物料
3. 一键生成入库单（草稿状态）
4. 选择库房和库位，登账入库
5. 登账后自动更新库房和库位的库存数量

#### 核心代码说明

**一键生成入库单核心代码**：

```java
// SpInboundOrderController.java - generateInternal方法
private Result generateInternal(String mrpIds, String warehouseId, String warehouseLocationId) {
    // 1. 解析选中的MRP记录ID列表
    List<String> selectedIds = new ArrayList<>();
    for (String id : mrpIds.split(",")) {
        if (StringUtils.isNotBlank(id)) {
            selectedIds.add(id.trim());
        }
    }
    
    // 2. 查询MRP记录
    List<SpMrpRecord> mrpRecords = new ArrayList<>(mrpRecordService.listByIds(selectedIds));
    
    // 3. 检查是否已生成入库单（防止重复生成）
    List<SpInboundOrderItem> items = new ArrayList<>();
    StringJoiner mrpNoJoiner = new StringJoiner(",");
    BigDecimal totalQty = BigDecimal.ZERO;
    
    for (SpMrpRecord mrpRecord : mrpRecords) {
        // 检查是否已存在入库单明细
        List<SpInboundOrderItem> matched = inboundOrderItemService.list(
            new QueryWrapper<SpInboundOrderItem>()
                .eq("source_mrp_record_id", mrpRecord.getId())
                .eq("is_deleted", "0")
        );
        if (!matched.isEmpty()) {
            return Result.failure("选中的物料需求存在已生成入库单的数据");
        }
        
        // 创建入库单明细
        SpInboundOrderItem item = new SpInboundOrderItem();
        item.setSourceMrpRecordId(mrpRecord.getId());
        item.setMrpNo(mrpRecord.getMrpNo());
        item.setDemandQty(mrpRecord.getDemandQty());
        // ... 设置其他字段
        items.add(item);
        
        totalQty = totalQty.add(mrpRecord.getDemandQty());
    }
    
    // 4. 生成入库单主表
    String inboundNo = "IB-" + LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyyMMddHHmmss"));
    SpInboundOrder order = new SpInboundOrder();
    order.setInboundNo(inboundNo);
    order.setStatus("draft");
    order.setItemCount(items.size());
    order.setTotalDemandQty(totalQty);
    inboundOrderService.save(order);
    
    // 5. 批量保存入库单明细
    for (SpInboundOrderItem item : items) {
        item.setInboundOrderId(order.getId());
    }
    inboundOrderItemService.saveBatch(items);
    
    // 6. 如果提供了库房信息，直接登账
    if (StringUtils.isNotBlank(warehouseId)) {
        applyInventory(order, items, warehouseId, warehouseLocationId);
        order.setStatus("confirmed");
        inboundOrderService.updateById(order);
    }
    
    return Result.success(result, "计划入库单生成成功");
}
```

**设计要点**：
- 使用`StringJoiner`拼接MRP编号，便于追溯
- 使用`BigDecimal`处理数量，避免精度问题
- 防止重复生成入库单的检查机制
- 支持两步操作：先生成草稿，再登账；或者一步生成并登账

**库存更新核心代码**：

```java
// SpInboundOrderController.java - applyInventory方法
private void applyInventory(SpInboundOrder order, List<SpInboundOrderItem> items, 
                           String warehouseId, String warehouseLocationId) {
    // 1. 计算入库总数量
    BigDecimal inboundQty = BigDecimal.ZERO;
    for (SpInboundOrderItem item : items) {
        if (item.getDemandQty() != null) {
            inboundQty = inboundQty.add(item.getDemandQty());
        }
    }
    
    // 2. 更新库房总库存
    SpWarehouse warehouse = warehouseService.getById(warehouseId);
    if (warehouse != null) {
        BigDecimal currentWarehouseInventory = 
            warehouse.getTotalInventory() == null ? BigDecimal.ZERO : warehouse.getTotalInventory();
        warehouse.setTotalInventory(currentWarehouseInventory.add(inboundQty));
        warehouseService.updateById(warehouse);
    }
    
    // 3. 更新库位库存（支持多个库位）
    if (StringUtils.isNotBlank(warehouseLocationId)) {
        BigDecimal remaining = inboundQty;
        List<String> usedLocationIds = new ArrayList<>();
        
        for (String locationId : warehouseLocationId.split(",")) {
            if (StringUtils.isBlank(locationId) || remaining.compareTo(BigDecimal.ZERO) <= 0) {
                continue;
            }
            
            SpWarehouseLocation location = warehouseLocationService.getById(locationId.trim());
            if (location == null) continue;
            
            // 计算库位可用容量
            BigDecimal currentLocationInventory = 
                location.getCurrentInventory() == null ? BigDecimal.ZERO : location.getCurrentInventory();
            BigDecimal maxCapacity = 
                location.getMaxCapacity() == null ? BigDecimal.ZERO : location.getMaxCapacity();
            BigDecimal freeCapacity = maxCapacity.subtract(currentLocationInventory);
            
            if (freeCapacity.compareTo(BigDecimal.ZERO) <= 0) {
                continue;
            }
            
            // 分配库存到库位
            BigDecimal allocated = remaining.min(freeCapacity);
            location.setCurrentInventory(currentLocationInventory.add(allocated));
            warehouseLocationService.updateById(location);
            remaining = remaining.subtract(allocated);
            usedLocationIds.add(location.getId());
        }
        
        // 4. 如果还有剩余，放入第一个库位（超额存储）
        if (remaining.compareTo(BigDecimal.ZERO) > 0 && !usedLocationIds.isEmpty()) {
            SpWarehouseLocation location = warehouseLocationService.getById(usedLocationIds.get(0));
            if (location != null) {
                location.setCurrentInventory(
                    location.getCurrentInventory().add(remaining));
                warehouseLocationService.updateById(location);
            }
        }
    }
}
```

**设计要点**：
- 库房库存 = 所有库位库存的总和
- 支持多库位分配，按顺序填满
- 超额存储允许（库位满了仍可以入库）
- 使用`BigDecimal`确保数量计算精度

**库位推荐核心代码**：

```java
// SpInboundOrderController.java - warehouseLocationSuggest方法
@GetMapping("/warehouse-location-suggest")
@ResponseBody
public Result warehouseLocationSuggest(String warehouseId, BigDecimal demandQty) {
    // 1. 查询库房下所有库位
    List<SpWarehouseLocation> locations = warehouseLocationService.list(
        new QueryWrapper<SpWarehouseLocation>()
            .eq("warehouse_id", warehouseId)
            .eq("is_deleted", "0")
            .orderByAsc("row_no", "col_no")
    );
    
    // 2. 计算每个库位的可用容量
    List<Map<String, Object>> candidates = new ArrayList<>();
    BigDecimal remaining = demandQty == null ? BigDecimal.ZERO : demandQty;
    
    for (SpWarehouseLocation location : locations) {
        BigDecimal current = location.getCurrentInventory() == null ? BigDecimal.ZERO : location.getCurrentInventory();
        BigDecimal max = location.getMaxCapacity() == null ? BigDecimal.ZERO : location.getMaxCapacity();
        BigDecimal free = max.subtract(current);
        
        if (free.compareTo(BigDecimal.ZERO) <= 0) {
            continue;  // 跳过已满库位
        }
        
        // 添加到候选列表
        Map<String, Object> item = new HashMap<>();
        item.put("id", location.getId());
        item.put("freeCapacity", free);
        // ... 其他字段
        candidates.add(item);
        
        remaining = remaining.subtract(free);
    }
    
    // 3. 按可用容量降序排序
    candidates.sort(Comparator.comparing(
        (Map<String, Object> m) -> new BigDecimal(m.get("freeCapacity").toString())
    ).reversed());
    
    // 4. 返回推荐结果
    Map<String, Object> result = new HashMap<>();
    result.put("candidates", candidates);
    result.put("canFulfill", demandQty == null || remaining.compareTo(BigDecimal.ZERO) <= 0);
    return Result.success(result);
}
```

**设计要点**：
- 智能推荐可用库位，按容量排序
- 计算是否能满足需求数量
- 支持前端展示库位使用情况

```java
// SpProductBomController.java - 获取BOM明细树
@GetMapping("/item/tree")
@ResponseBody
public Result getBomTree(String bomId) {
    // 1. 查询BOM下所有明细
    List<SpProductBomItem> items = bomItemService.list(
        new QueryWrapper<SpProductBomItem>()
            .eq("bom_id", bomId)
            .eq("is_deleted", "0")
            .orderByAsc("level_no", "line_no")
    );
    
    // 2. 构建树形结构
    List<TreeNode> tree = buildTree(items);
    return Result.success(tree);
}

// 构建树形结构的辅助方法
private List<TreeNode> buildTree(List<SpProductBomItem> items) {
    Map<String, TreeNode> nodeMap = new HashMap<>();
    List<TreeNode> roots = new ArrayList<>();
    
    // 创建所有节点
    for (SpProductBomItem item : items) {
        TreeNode node = new TreeNode(item);
        nodeMap.put(item.getId(), node);
    }
    
    // 建立父子关系
    for (SpProductBomItem item : items) {
        TreeNode node = nodeMap.get(item.getId());
        if (item.getParentItemId() == null) {
            // 根节点
            roots.add(node);
        } else {
            // 子节点
            TreeNode parent = nodeMap.get(item.getParentItemId());
            if (parent != null) {
                parent.getChildren().add(node);
            }
        }
    }
    
    return roots;
}
```

**设计要点**：
- 使用Map缓存所有节点，避免重复查询
- 通过`parent_item_id`建立父子关系
- 根节点的`parent_item_id`为null
- 按`level_no`和`line_no`排序保证顺序正确

**BOM循环引用检测核心代码**：

```java
// 添加BOM明细时的循环引用检测
private boolean checkCircularReference(String bomId, String parentItemId, String partCode) {
    // 检测逻辑：从当前节点向上遍历，如果partCode出现在路径中，则存在循环引用
    Set<String> visited = new HashSet<>();
    String current = parentItemId;
    
    while (current != null) {
        SpProductBomItem item = bomItemService.getById(current);
        if (item == null) break;
        
        if (item.getPartCode().equals(partCode)) {
            return true; // 发现循环引用
        }
        
        if (visited.contains(current)) {
            return true; // 防止死循环
        }
        visited.add(current);
        current = item.getParentItemId();
    }
    
    return false;
}
```

**设计要点**：
- 防止BOM中出现循环引用（A的子节点是B，B的子节点又是A）
- 使用visited集合防止死循环
- 从父节点向上遍历检测

```java
@PostMapping("/person-dispatch")
@ResponseBody
public Result personDispatch(@RequestBody Map<String, Object> params) {
    String orderId = (String) params.get("orderId");
    String userId = (String) params.get("userId");
    String processUnitId = (String) params.get("processUnitId");
    Object qtyObj = params.get("qty");

    // 1. 参数校验
    if (StringUtils.isBlank(orderId) || StringUtils.isBlank(userId) || StringUtils.isBlank(processUnitId)) {
        return Result.failure("参数不能为空");
    }

    // 2. 查询派工单
    SpDispatchOrder order = dispatchOrderService.getOne(
            new QueryWrapper<SpDispatchOrder>()
                    .eq("id", orderId)
                    .eq("is_deleted", "0"));
    if (order == null) {
        return Result.failure("工单不存在");
    }

    // 3. 状态校验：只有草稿和已分配状态才能派工
    if (!"draft".equals(order.getStatus()) && !"assigned".equals(order.getStatus())) {
        return Result.failure("当前工单状态不允许派工");
    }

    // 4. 查询用户和加工单元
    SysUser user = userService.getById(userId);
    SysProcessUnit processUnit = processUnitService.getById(processUnitId);

    // 5. 创建派工记录
    BigDecimal qty = new BigDecimal(qtyObj != null ? qtyObj.toString() : order.getQty().toString());
    SpDispatchRecord record = new SpDispatchRecord();
    record.setDispatchOrderId(orderId);
    record.setDispatchType("person");  // 人员派工
    record.setProcessUnitId(processUnitId);
    record.setTeamId(processUnit.getGroupId());
    record.setUserId(userId);
    record.setPlanQty(qty);
    record.setDispatchTime(LocalDateTime.now());
    record.setStatus("pending");  // 待开工
    dispatchRecordService.save(record);

    // 6. 更新派工单状态为已分配
    order.setStatus("assigned");
    dispatchOrderService.updateById(order);

    return Result.success();
}
```

**设计要点**：
- 使用`@RequestBody Map`接收参数，灵活处理前端数据
- 状态校验防止非法操作
- 派工记录状态为"pending"，等待员工开工
- 派工单状态变为"assigned"，表示已有任务分配

**完工核心代码**：

```java
// SpDispatchController.java - completeWork方法
@PostMapping("/complete-work")
@ResponseBody
public Result completeWork(@RequestBody Map<String, Object> params) {
    String recordId = (String) params.get("recordId");
    Object qualifiedObj = params.get("qualifiedQty");
    Object scrapObj = params.get("scrapQty");

    // 1. 查询派工记录
    SpDispatchRecord record = dispatchRecordService.getOne(
            new QueryWrapper<SpDispatchRecord>()
                    .eq("id", recordId)
                    .eq("is_deleted", "0"));

    // 2. 状态校验：只有started状态才能完工
    if (!"started".equals(record.getStatus())) {
        return Result.failure("当前状态不允许完工");
    }

    // 3. 计算完工数量
    BigDecimal qualifiedQty = new BigDecimal(qualifiedObj != null ? qualifiedObj.toString() : "0");
    BigDecimal scrapQty = new BigDecimal(scrapObj != null ? scrapObj.toString() : "0");
    BigDecimal completedQty = qualifiedQty.add(scrapQty);

    // 4. 更新派工记录
    record.setCompletedQty(completedQty);
    record.setQualifiedQty(qualifiedQty);
    record.setScrapQty(scrapQty);
    record.setStatus("completed");
    record.setEndTime(LocalDateTime.now());
    
    // 5. 计算工时（分钟转小时）
    if (record.getStartTime() != null) {
        long hours = java.time.Duration.between(record.getStartTime(), record.getEndTime()).toMinutes();
        record.setWorkHours(new BigDecimal(hours).divide(new BigDecimal(60), 2, BigDecimal.ROUND_HALF_UP));
    }
    dispatchRecordService.updateById(record);

    // 6. 更新派工单的累计数量
    SpDispatchOrder order = dispatchOrderService.getOne(
            new QueryWrapper<SpDispatchOrder>()
                    .eq("id", record.getDispatchOrderId())
                    .eq("is_deleted", "0"));
    if (order != null) {
        order.setCompletedQty(order.getCompletedQty().add(completedQty));
        order.setQualifiedQty(order.getQualifiedQty().add(qualifiedQty));
        order.setScrapQty(order.getScrapQty().add(scrapQty));

        // 7. 判断派工单是否全部完成
        if (order.getCompletedQty().compareTo(order.getQty()) >= 0) {
            order.setStatus("completed");  // 全部完成
            order.setActualEndTime(LocalDateTime.now());
        } else {
            order.setStatus("inspected");  // 部分完工
        }
        dispatchOrderService.updateById(order);
    }

    return Result.success();
}
```

**设计要点**：
- 完工数量 = 合格品数 + 报废品数
- 自动计算工时（结束时间 - 开始时间）
- 累计更新派工单的完成数量
- 判断是否全部完成，更新派工单状态
- 使用`BigDecimal`确保数量计算精度

```java
// 计量型检验合格判定
public boolean isQualified(BigDecimal actualValue, BigDecimal standardValue, 
                           BigDecimal upperTolerance, BigDecimal lowerTolerance) {
    BigDecimal upperLimit = standardValue.add(upperTolerance);
    BigDecimal lowerLimit = standardValue.subtract(lowerTolerance);
    
    return actualValue.compareTo(lowerLimit) >= 0 
        && actualValue.compareTo(upperLimit) <= 0;
}
```

**设计要点**：
- 实测值在[标准值-下允差, 标准值+上允差]范围内判定为合格
- 使用BigDecimal确保精度
- 记录检验时间和检验员，便于追溯

```java
// SpOrderServiceImpl.java - execute方法
@Transactional(rollbackFor = Exception.class)
public Result execute(String orderId) {
    // 1. 查询订单
    SpOrder order = getById(orderId);
    if (order == null) {
        return Result.failure("订单不存在");
    }
    
    // 2. 状态校验
    if (!"approved".equals(order.getStatue())) {
        return Result.failure("只有已审核的订单才能执行");
    }
    
    // 3. 根据BOM生成MRP需求
    String mrpNo = generateMrp(order);
    order.setGeneratedMrpNo(mrpNo);
    
    // 4. 根据工艺路线生成派工单
    String planNo = generateDispatchPlan(order);
    order.setGeneratedPlanNo(planNo);
    
    // 5. 更新订单状态
    order.setStatue("executing");
    updateById(order);
    
    return Result.success();
}

// 根据BOM生成MRP需求
private String generateMrp(SpOrder order) {
    // 1. 查询产品的BOM
    SpProductBom bom = bomService.getOne(
        new QueryWrapper<SpProductBom>()
            .eq("product_part_code", order.getMateriel())
            .eq("state", "released")  // 只使用已发布的BOM
    );
    
    // 2. 递归展开BOM，计算物料需求
    List<SpMrpRecord> mrpRecords = new ArrayList<>();
    expandBom(bom.getId(), order.getQty(), mrpRecords, order.getOrderCode());
    
    // 3. 批量保存MRP记录
    mrpRecordService.saveBatch(mrpRecords);
    
    // 4. 返回MRP编号
    return "MRP" + LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyyMMddHHmmss"));
}

// 递归展开BOM
private void expandBom(String bomId, BigDecimal orderQty, 
                       List<SpMrpRecord> mrpRecords, String orderCode) {
    // 查询BOM明细
    List<SpProductBomItem> items = bomItemService.list(
        new QueryWrapper<SpProductBomItem>()
            .eq("bom_id", bomId)
            .eq("is_deleted", "0")
    );
    
    for (SpProductBomItem item : items) {
        // 计算需求数量
        BigDecimal demandQty = orderQty.multiply(item.getQty());
        if (item.getLossRate() != null) {
            demandQty = demandQty.multiply(
                new BigDecimal(1).add(item.getLossRate()));
        }
        
        // 创建MRP记录
        SpMrpRecord mrpRecord = new SpMrpRecord();
        mrpRecord.setPartCode(item.getPartCode());
        mrpRecord.setPartName(item.getPartName());
        mrpRecord.setDemandQty(demandQty);
        mrpRecord.setUnit(item.getUnit());
        mrpRecord.setSourceOrderNo(orderCode);
        mrpRecords.add(mrpRecord);
        
        // 如果该物料还有子BOM，递归展开
        SpProductBom subBom = bomService.getOne(
            new QueryWrapper<SpProductBom>()
                .eq("product_part_code", item.getPartCode())
                .eq("state", "released")
        );
        if (subBom != null) {
            expandBom(subBom.getId(), demandQty, mrpRecords, orderCode);
        }
    }
}
```

**设计要点**：
- 使用事务确保订单执行的原子性
- BOM递归展开计算物料需求
- 考虑损耗率，计算准确的物料需求
- 只使用已发布（锁定）的BOM
- 使用BigDecimal确保数量计算精度

##### 7.2.1 录入生产订单

**功能入口**：计划管理 > 生产订单 > 新增

**业务流程**：
1. 计划员点击新增生产订单
2. 系统自动生成订单编号
3. 填写订单基本信息：
   - 产品信息（选择产品物料）
   - 生产数量
   - 计划开始/完成日期
   - 客户信息（可选）
   - 销售订单号（可选）
   - 优先级设置
4. 提交订单
5. 订单状态变为"草稿"
6. 提交审核，状态变为"待审核"

**订单状态流转**：
```
草稿 → 待审核 → [驳回] → 草稿
          ↓
        已审核 → 已执行 → 完成
```

---

##### 7.2.2 自动生成作业工单

**触发条件**：生产订单审核通过后，点击"执行"按钮。

**生成逻辑**：
1. 根据产品BOM和工艺路线
2. 按工序分解为多个作业工单（派工单）
3. 计算各工序的计划时间（考虑日历和工作日）
4. 分配到对应的加工单元
5. 派工单初始状态为"草稿"

---

##### 7.2.3 自动计算物料需求

**计算逻辑**：
1. 基于BOM结构展开计算
2. 计算公式：毛需求 = 订单数量 × BOM用量 × (1 + 损耗率)
3. 净需求 = 毛需求 - 现有库存 - 在途库存
4. 生成物料需求清单（MRP记录）

**数据展示**：
- 显示每个物料的毛需求、可用库存、在途库存、净需求
- 建议采购/生产数量
- 需求日期（根据工序倒推）
