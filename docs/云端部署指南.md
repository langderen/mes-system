# MES系统云端部署指南

## 📦 项目打包信息

- **项目名称**: MES制造执行系统
- **版本**: 1.0.0
- **打包文件**: `mes-1.0.0.jar`
- **位置**: `target/mes-1.0.0.jar`

## 🚀 部署方式一：Docker容器部署（推荐）

### 1. 准备环境

确保你的服务器已安装：
- Docker
- Docker Compose（可选，推荐）

### 2. 使用Dockerfile构建镜像

```bash
# 进入项目根目录
cd mes

# 构建Docker镜像
docker build -t mes-system:1.0.0 -f src/main/docker/Dockerfile .
```

### 3. 运行Docker容器

```bash
# 方式一：直接运行（不推荐生产环境）
docker run -d --name mes-system \
  -p 80:9090 \
  -v /data/mes-upload:/upload \
  mes-system:1.0.0

# 方式二：使用环境变量配置（推荐）
docker run -d --name mes-system \
  -p 80:9090 \
  -e SPRING_PROFILES_ACTIVE=pro \
  -e SPRING_DATASOURCE_URL=jdbc:mysql://your-mysql-host:3306/mes2026 \
  -e SPRING_DATASOURCE_USERNAME=your-username \
  -e SPRING_DATASOURCE_PASSWORD=your-password \
  -e SPRING_REDIS_HOST=your-redis-host \
  -v /data/mes-upload:/upload \
  --restart=always \
  mes-system:1.0.0
```

### 4. 使用Docker Compose（推荐生产环境）

创建 `docker-compose.yml` 文件：

```yaml
version: '3.8'
services:
  mes-system:
    image: mes-system:1.0.0
    container_name: mes-system
    ports:
      - "80:9090"
    environment:
      - SPRING_PROFILES_ACTIVE=pro
      - SPRING_DATASOURCE_URL=jdbc:mysql://mysql:3306/mes2026?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
      - SPRING_DATASOURCE_USERNAME=root
      - SPRING_DATASOURCE_PASSWORD=your-mysql-password
      - SPRING_REDIS_HOST=redis
      - SPRING_REDIS_PORT=6379
      - FILE_UPLOAD_DIR=/upload
    volumes:
      - mes-upload:/upload
    depends_on:
      - mysql
      - redis
    restart: always
    networks:
      - mes-network

  mysql:
    image: mysql:5.7
    container_name: mes-mysql
    environment:
      - MYSQL_ROOT_PASSWORD=your-mysql-password
      - MYSQL_DATABASE=mes2026
      - MYSQL_USER=mes
      - MYSQL_PASSWORD=your-mysql-password
      - TZ=Asia/Shanghai
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql
      - ./scripts/sql:/docker-entrypoint-initdb.d
    restart: always
    networks:
      - mes-network
    command: --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci

  redis:
    image: redis:6-alpine
    container_name: mes-redis
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    restart: always
    networks:
      - mes-network

volumes:
  mes-upload:
  mysql-data:
  redis-data:

networks:
  mes-network:
    driver: bridge
```

启动服务：

```bash
docker-compose up -d
```

## 🚀 部署方式二：云服务器直接部署

### 1. 环境准备

确保服务器安装：
- JDK 8 或更高版本
- MySQL 5.7+
- Redis（可选，推荐）

### 2. 上传JAR文件

将 `target/mes-1.0.0.jar` 上传到服务器，例如上传到 `/opt/mes/` 目录。

### 3. 准备数据库

```bash
# 导入数据库脚本
mysql -u root -p mes2026 < scripts/sql/20260607124109.sql
```

### 4. 创建启动脚本

创建 `/opt/mes/start.sh`：

```bash
#!/bin/bash
APP_NAME="mes-1.0.0.jar"
APP_HOME="/opt/mes"
LOG_FILE="$APP_HOME/logs/mes.log"

# 创建日志目录
mkdir -p $APP_HOME/logs

# 停止已运行的进程
PID=$(ps -ef | grep $APP_NAME | grep -v grep | awk '{print $2}')
if [ -n "$PID" ]; then
    echo "Stopping existing process..."
    kill -9 $PID
fi

# 启动应用
echo "Starting MES system..."
nohup java -jar $APP_HOME/$APP_NAME \
  --spring.profiles.active=pro \
  --server.port=9090 \
  > $LOG_FILE 2>&1 &

echo "MES system started! Check logs at $LOG_FILE"
```

### 5. 配置application-pro.yml

创建 `/opt/mes/config/application-pro.yml`：

```yaml
server:
  port: 9090

spring:
  mvc:
    view:
      prefix: /templates/
      suffix: .ftl
      static-path-pattern: /static/**
  freemarker:
    suffix: .ftl
    templateEncoding: UTF-8
    templateLoaderPath: classpath:/templates/
    cache: true
    expose-spring-macro-helpers: false
    request-context-attribute: request
    settings:
      classic_compatible: true
  datasource:
    type: com.alibaba.druid.pool.DruidDataSource
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://your-mysql-host:3306/mes2026?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: your-username
    password: your-password
    druid:
      initial-size: 8
      min-idle: 5
      max-active: 20
      query-timeout: 6000
      transaction-query-timeout: 6000
      remove-abandoned-timeout: 1800
      filters: stat,config
  redis:
    host: your-redis-host
    port: 6379
    password:
    timeout: 10000
    jedis:
      pool:
        max-idle: 8
        min-idle: 10
        max-active: 100
        max-wait: -1
  servlet:
    multipart:
      max-file-size: 10MB
      max-request-size: 10MB

logging:
  level:
    root: info
    com.wangziyang: info
  file:
    name: /opt/mes/logs/mes.log

file:
  upload-dir: /opt/mes-upload
  access-path: /upload
```

### 6. 启动服务

```bash
# 赋予执行权限
chmod +x /opt/mes/start.sh

# 启动服务
/opt/mes/start.sh

# 查看日志
tail -f /opt/mes/logs/mes.log
```

### 7. 使用systemd管理服务（推荐生产环境）

创建 `/etc/systemd/system/mes.service`：

```ini
[Unit]
Description=MES System Service
After=network.target mysql.service redis.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/mes
ExecStart=/usr/bin/java -jar /opt/mes/mes-1.0.0.jar --spring.profiles.active=pro
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

启用并启动服务：

```bash
# 重载systemd配置
systemctl daemon-reload

# 启用开机自启
systemctl enable mes.service

# 启动服务
systemctl start mes.service

# 查看状态
systemctl status mes.service

# 查看日志
journalctl -u mes.service -f
```

## 🚀 部署方式三：云平台部署

### 阿里云部署

1. 登录阿里云容器服务（ACK）
2. 创建Kubernetes集群
3. 编写K8s部署文件 `k8s-deployment.yaml`：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: mes-system
  labels:
    app: mes-system
spec:
  replicas: 2
  selector:
    matchLabels:
      app: mes-system
  template:
    metadata:
      labels:
        app: mes-system
    spec:
      containers:
      - name: mes-system
        image: mes-system:1.0.0
        ports:
        - containerPort: 9090
        env:
        - name: SPRING_PROFILES_ACTIVE
          value: "pro"
        - name: SPRING_DATASOURCE_URL
          valueFrom:
            secretKeyRef:
              name: mes-secret
              key: datasource-url
        - name: SPRING_DATASOURCE_USERNAME
          valueFrom:
            secretKeyRef:
              name: mes-secret
              key: datasource-username
        - name: SPRING_DATASOURCE_PASSWORD
          valueFrom:
            secretKeyRef:
              name: mes-secret
              key: datasource-password
        resources:
          requests:
            memory: "512Mi"
            cpu: "500m"
          limits:
            memory: "1Gi"
            cpu: "1000m"
        livenessProbe:
          httpGet:
            path: /
            port: 9090
          initialDelaySeconds: 60
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /
            port: 9090
          initialDelaySeconds: 30
          periodSeconds: 5

---
apiVersion: v1
kind: Service
metadata:
  name: mes-system-service
spec:
  type: LoadBalancer
  selector:
    app: mes-system
  ports:
  - protocol: TCP
    port: 80
    targetPort: 9090

---
apiVersion: v1
kind: Secret
metadata:
  name: mes-secret
type: Opaque
data:
  datasource-url: base64-encoded-url
  datasource-username: base64-encoded-username
  datasource-password: base64-encoded-password
```

4. 部署到K8s：

```bash
kubectl apply -f k8s-deployment.yaml
```

## 🔧 常见问题

### 1. 端口被占用

```bash
# 查看端口占用
netstat -tulpn | grep 9090

# 修改配置文件中的端口号
```

### 2. 数据库连接失败

- 检查数据库服务是否启动
- 确认连接地址、用户名、密码正确
- 检查防火墙设置

### 3. 文件上传失败

- 检查上传目录权限
- 确保配置的目录存在且可写

### 4. 内存不足

```bash
# 增加JVM内存
java -Xms512m -Xmx1024m -jar mes-1.0.0.jar
```

## 📊 监控与运维

### 查看应用日志

```bash
# Docker方式
docker logs -f mes-system

# 直接部署方式
tail -f /opt/mes/logs/mes.log

# systemd方式
journalctl -u mes.service -f
```

### 备份数据库

```bash
# 备份
mysqldump -u root -p mes2026 > backup_$(date +%Y%m%d).sql

# 恢复
mysql -u root -p mes2026 < backup_20260607.sql
```

### 更新应用

```bash
# 1. 上传新的JAR文件
# 2. 停止服务
systemctl stop mes.service
# 3. 备份旧版本
mv /opt/mes/mes-1.0.0.jar /opt/mes/mes-1.0.0.jar.bak
# 4. 替换新版本
cp mes-1.0.0-new.jar /opt/mes/mes-1.0.0.jar
# 5. 启动服务
systemctl start mes.service
```
